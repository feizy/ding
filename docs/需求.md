# `ding` 当前需求摘要

更新时间：2026-04-10

`ding` 通过 CLI 启动 Claude Code / Codex / 自定义程序，使用 Always-on-Top 悬浮窗实时展示实例状态；在 Agent 需要人工审批时，通过桌面浮窗让用户快速确认后继续执行。

当前最高优先级需求：

- 执行 `ding claude` 后，当前 CLI 中显示的必须是 Claude Code 原生交互式 TUI
- 该 TUI 的行为必须与“在当前目录直接执行 `claude`”完全一致
- `ding` 对 Claude 的监控只依赖官方 hooks，不再依赖 `claude -p --output-format stream-json`
- `ding` 允许并负责修改用户级 Claude 配置文件，使 hooks 安装在 `~/.claude/settings.json`
- 安装完成后，之后直接执行 `claude` 也默认带上 `ding` hooks
